EVE Chat v1.3.0
================

本版本以 v1.2 为基础，主要完成两项调整：

1. 角色行程表成为主屏幕独立 App
2. 加入“记忆待确认”功能（v1.2 尚未加入的项目）


一、主屏幕独立行程 App
----------------------

主屏幕右上区域会从原本的两个入口改成 2×2 排列：

Chat       世界书
行程       待确认

点“行程”会进入独立页面，不再只能从聊天设置中的弹窗查看。

行程 App 支持：
- 前一天／今天／后一天切换
- 显示当前安排与下一项安排
- 生成或重新生成当天行程
- 手动增加、编辑、删除行程
- 标记完成、恢复、锁定与解锁
- 打开原有详细管理器和行程设置
- 主屏幕角标显示当天尚未结束的安排数
- 按角色分别保存
- 保留天气、场景状态、R1、通知和时间线联动

设置入口仍保留：
聊天设置 → EVE 扩展功能 → 角色行程表


二、记忆待确认
--------------

主屏幕新增“待确认”入口。

升级后，新识别到的长期资料和重要事件不会立即写进正式长期记忆，而会先进入待确认页面，例如：
- 名字、生日、身份和居住地
- 明确表达的喜欢／不喜欢
- 重要目标
- 明天、后天或近期计划
- 已发生的重要事件

每条建议可以：
- 修改
- 忽略
- 保留并写入长期记忆
- 全部保留
- 全部忽略

每条建议会保留来源消息 ID。撤回原消息时，仍未确认的对应建议也会一并删除。

为了避免重复写入，启用“记忆待确认”后，EVEMemory 原本的长期事实／事件自动提取会暂停；最近对话片段（conversation moments）仍会正常保存，用于维持上下文。

过去版本已经写入的记忆不会被删除。此规则只影响升级后的新消息。

如关闭“记忆待确认”，系统会恢复升级前的自动提取设置。

设置入口：
聊天设置 → EVE 扩展功能 → 记忆待确认


三、部署
--------

将压缩包内的所有内容上传到 GitHub EVE 仓库根目录并覆盖旧文件。

根目录必须直接看到：
- index.html
- eve-sw.js
- js/
- plugins/
- character-packs/

特别确认这些新文件存在：
- plugins/daily-schedule/app.js
- plugins/memory-inbox/core.js

部署完成后建议使用：
https://kanamememe.github.io/EVE/?v=1.3.0

避免 Safari 读取旧版缓存。


四、不包含的功能
----------------

本版本没有加入关系值、好感度、信任度或自动关系阶段系统。
