EVE Chat v1.3.0
主页独立行程 App／记忆待确认匣
============================================

本版本以 EVE Chat v1.2 为基础，保留原有：
- 聊天、动态、世界书、短信、纪念日、论坛
- 天气、主动聊天、扩展记忆、共同时间线
- 消息撤回与关联记忆／时间线清理
- 批量表情包、表情包智能识别
- 当前场景状态、AI限制诊断
- 通用角色贴合引擎
- 萧逸・夕月专属角色包、R1赛事与专属行程规则

本版新增／修改
------------------------------------------------------------

一、主页独立“行程”App

“行程”不再只藏在聊天设置里。

主页左下角原本的“纪念日／论坛”区域会改为 2×2 小应用网格，并新增：

  行程

它和 Chat、纪念日一样，是可以直接从主页打开的独立 App。
原本的纪念日和论坛不会删除。

进入“行程”后可以：
- 查看角色今天、前一天和后一天的行程
- 查看当前正在做什么、下一项是什么
- 一键生成某一天的行程
- 手动新增行程
- 编辑时间、标题、说明、私密状态
- 切换“计划中／进行中／已完成／已调整／已取消／错过”
- 删除单项行程
- 清空某一天的行程
- 进入原有行程设置
- 查看主页红色角标中的未完成／进行中项目数量

行程仍然按角色分开保存。
如果还没有在 Chat 中打开角色，行程页会提示先选择角色。

行程生成模式仍支持：
- 整日规划
- 到点生成
- 混合模式

萧逸角色包仍会参考：
- 晨跑、体能与反应训练
- 模拟器训练、赛车调校、车队会议、遥测复盘
- LONGDAY任务、机车、做饭、朋友与宠物
- R1／F1赛历原型和比赛周末 Session

当前明确聊天场景仍然高于预设行程，避免角色同时出现在两个地点。

二、记忆待确认匣

v1.2 尚未加入的“记忆待确认匣”已在本版补上。

启用确认模式后：
- AI 从使用者消息中识别出的长期事实与事件，不会立刻写入正式记忆
- 候选内容先进入“待确认记忆”
- 你可以保留、修改或忽略
- 可以一次全部保留或全部忽略
- 手动确认后才会写入扩展记忆
- 原本的对话片段记录仍可继续自动保存

可能进入待确认匣的内容包括：
- 姓名
- 生日
- 喜欢或讨厌的事物
- 居住地
- 身份、工作或学业
- 目标和计划
- 明天、今天、最近发生的重要事件

入口有两个：

1. 聊天设置 → EVE 扩展功能 → 待确认记忆
2. 主页“记忆”Dock 图标上的红色数字角标

在扩展记忆管理器中，也会出现“待确认”入口。

撤回同步：
- 待确认候选会记录来源消息 ID
- 如果来源消息被撤回，对应尚未确认的候选会一并删除
- 已经确认的正式记忆仍由原有撤回系统按来源 ID 清理

确认模式开启时，扩展记忆的直接自动提取会关闭，避免同一内容一边进入待确认匣、一边又被直接保存。

设置位置
------------------------------------------------------------

聊天设置 → EVE 扩展功能

可看到：
- 扩展记忆
- 待确认记忆
- 角色行程表

“角色行程表”仍保留开关；点击这一行会直接打开主页同款独立行程 App。

部署
------------------------------------------------------------

1. 解压 ZIP
2. 把 ZIP 里面的内容全部上传到 GitHub EVE 仓库根目录
3. 必须一起覆盖 index.html 和以下目录：

   js/
   plugins/
   character-packs/

4. 特别确认新增文件存在：

   plugins/daily-schedule/app.js
   plugins/memory-inbox/core.js

5. 也要覆盖：

   js/feature-settings.js
   js/healthcheck.js
   index.html

6. 部署后用以下网址避开 Safari 快取：

   https://kanamememe.github.io/EVE/?v=1.3.0

限制
------------------------------------------------------------

- 网页在 iOS 后台仍可能暂停；重新打开时由现有行程模块补检查
- 真实 Gemini、Open-Meteo、Jolpica、OpenF1 仍需部署后联网实测
- 记忆候选使用本地规则提取，复杂语义仍可能需要人工修改后保存
- 本版本没有关系值／好感度系统
