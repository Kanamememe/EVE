EVE Chat v1.2.0
表情包智能识别 v2／当前场景状态／角色行程表／AI限制诊断
============================================================

本版本以 EVEChat v1.1.2 为基础，保留原有：
- 聊天、世界书、动态、短信、记忆、时间线
- 消息引用与 AI 引用修复
- 批量表情包导入与图片显示修复
- 天气、主动聊天、通知、网页图标
- 通用角色贴合引擎
- 萧逸・夕月专属角色包与 R1 实时赛事

本版新增／加强
------------------------------------------------------------

一、表情包智能识别 v2

设置位置：
聊天设置 → EVE 扩展功能 → 表情包智能识别

功能：
- 为表情包保存画面描述、图片文字、情绪、聊天意图、适用场景、语气、主体、动作与强度
- 理解使用者刚发送的表情包，并把含义与置信度提供给 AI
- 根据最新对话、场景、角色偏好与最近使用记录筛选表情包候选
- AI 使用稳定 stickerId 发送，不再只依赖 IMG_XXXX 等文件名
- 可使用当前 Gemini API 对表情包做一次性视觉分析
- GIF 在浏览器支持 ImageDecoder 时会抽取多帧分析；不支持时回退到首帧
- 可手动纠正含义并锁定，避免以后被 AI 覆盖
- 萧逸模式会偏好调侃、得意、看戏、无奈、猫猫等表情，减少卑微大哭或过度卖萌

注意：
- “AI分析未识别”会消耗 Gemini API 额度
- 自动分析默认关闭
- 低置信度时只把图片当作情绪性回应，不会强行断言使用者的具体意图

二、当前场景状态

设置位置：
聊天设置 → EVE 扩展功能 → 当前场景状态

功能：
- 保存当前互动形式：文字聊天、面对面、语音、视频等
- 保存当前地点、在场人物、正在做的事、当前事件
- 保存尚未完成的动作和本场景临时事实
- 从聊天中自动识别部分地点、活动、等待事项
- 防止“明明面对面却说我过去接你”等地点和动作冲突
- 当前明确聊天情境优先于预设行程
- 可手动查看、修改与锁定场景字段

三、角色行程表

设置位置：
聊天设置 → EVE 扩展功能 → 角色行程表

生成模式：
- 整日规划：一次生成当天完整行程
- 到点生成：当前时段到来时只生成下一项行程
- 混合模式：先生成全天骨架，到点时可补充细节

功能：
- 按角色当地时间生成
- 可参考天气、扩展记忆、时间线、原有约定与 R1 赛事
- 行程状态：计划中、进行中、已完成、已调整、已取消、错过
- 可以手动新增、编辑、删除和锁定行程
- 行程开始时可写入时间线、显示通知或允许角色主动发消息
- Safari 暂停后，重新打开页面会补检查期间应该发生的行程
- 当前场景与行程冲突时，以最新明确场景为准，并要求 AI 自然调整行程

萧逸专属行程：
- 晨跑、体能与反应训练、模拟器训练
- 赛车调校、车队会议、遥测与赛后复盘
- LONGDAY 任务、机车、做饭、朋友、音乐与宠物
- R1 比赛周末优先同步练习、排位、冲刺、正赛等 Session
- 重要赛道活动前保留独处五分钟的习惯
- 现实 F1 仅作为 R1 赛历与赛事流程原型，不会自动决定萧逸的名次、伤势或退赛

四、AI 限制诊断

设置位置：
聊天设置 → EVE 扩展功能 → AI 限制诊断

记录：
- 模型名称
- HTTP 状态
- promptFeedback.blockReason
- finishReason
- safetyRatings
- Prompt safetyRatings
- Safety Settings
- API错误、网络错误、空回复与解析失败

用途：
帮助区分 Gemini 安全拦截、API 错误、空回应与 EVE 解析／显示问题。

部署
------------------------------------------------------------

1. 解压 ZIP
2. 把 ZIP 里面的内容上传到 GitHub EVE 仓库根目录
3. 必须保留这些目录：

   js/
   plugins/
   character-packs/

4. 特别确认以下新文件存在：

   plugins/sticker-intelligence/core.js
   plugins/scene-state/core.js
   plugins/daily-schedule/core.js
   js/ai-diagnostics.js
   character-packs/xiaoyi/schedule.js

5. 部署后用以下网址避开 Safari 快取：

   https://kanamememe.github.io/EVE/?v=1.2.0

限制
------------------------------------------------------------

- 真实 Gemini、Open-Meteo、Jolpica 与 OpenF1 联网仍需部署后实机验证
- 网页在 iOS 后台会被暂停；重新打开时会补算行程，但无法在完全关闭时即时调用 Gemini
- “记忆待确认匣”没有包含在本版，现有记忆提取方式保持不变
- 本版没有关系值／好感度系统
