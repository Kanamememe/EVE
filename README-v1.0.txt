EVE Chat v1.0 — 动态连续回复与网页图标
========================================

本版以用户已经确认可正常使用的 v0.9 为基础继续扩展。
保留 v0.9 的聊天、动态回复修复、通知、天气、主动聊天、记忆、时间线、撤回与表情包功能。
本版仍然不包含关系值、好感度、信任度或关系阶段系统。

一、部署
--------
1. 解压 ZIP。
2. 将解压后的 index.html、eve-sw.js、整个 js 文件夹和其他同级文件上传到 GitHub 仓库 EVE 根目录。
3. 不要只上传 index.html；js/web-icon.js 和更新后的 js/moments.js、js/feature-settings.js、js/healthcheck.js 必须一起上传。
4. 根目录应直接看到：

   EVE/
   ├─ index.html
   ├─ eve-sw.js
   ├─ js/
   ├─ original/
   ├─ README-v1.0.txt
   ├─ CHANGELOG-v1.0.txt
   └─ VALIDATION-v1.0.txt

5. 部署后建议用以下网址绕过旧缓存：
   https://kanamememe.github.io/EVE/?v=1.0

二、回复动态下角色的回答
----------------------
角色在动态评论区回答后，该条角色评论旁会出现一个小型“回复”按钮。
点击后，底部输入框上方会显示正在回复的角色和评论摘要。
发送后：

- 用户评论会记录它所回复的角色评论 ID。
- 角色的新回答会记录它所回复的用户评论 ID。
- 角色回答后仍可继续点击“回复”，进行多轮评论区对话。
- AI 会读取当前回复链，而不只看到最后一句，因此更不容易重复上一轮回答或答非所问。
- 原动态页面、评论保存方式和通知功能继续保留。

设置位置：
聊天设置 → EVE 扩展功能 → 动态回复增强

新增设置：
- 允许继续回复角色在评论区的回答
- 在角色评论旁显示“回复”按钮
- 连续回复最多读取评论数

说明：
- “连续回复最多读取评论数”建议设为 8～16。
- 关闭“继续回复”后，v0.9 原有的动态回复修复仍然可以继续使用。
- 若增强生成失败，可继续使用原生逻辑回退。

三、网页图标
------------
设置位置：
聊天设置 → EVE 扩展功能 → 网页图标

可以：
- 上传 PNG、JPG、JPEG、WebP 或 GIF 图片。
- 自动从图片中心裁切为正方形。
- 自动生成 32、180、192、512 像素 PNG 图标。
- 同时更新浏览器 favicon、shortcut icon、Apple touch icon 和运行时网页清单图标。
- 图标保存到 EVE Chat 现有 IndexedDB；IndexedDB 不可用时自动退回 localStorage。
- 随时恢复 EVE Chat 默认图标。

平台差异：
- 浏览器标签页图标通常会立即更新。
- Safari 可能需要重新载入页面才能刷新标签图标。
- iPhone 已经添加到主屏幕的图标由 iOS 缓存。更换后需要删除旧主屏幕捷径，再从 Safari 重新“添加到主屏幕”。
- Android 已安装的旧网页应用图标也可能需要重新安装才能更新。

五、建议测试顺序
----------------
1. 确认原本角色聊天正常。
2. 打开“扩展功能健康检查”，确认 moments、web-icon 项目已载入。
3. 打开一条有角色评论的动态。
4. 点击角色评论旁的“回复”。
5. 连续回复角色两轮，确认每一轮角色回答旁仍有“回复”按钮。
6. 打开“网页图标”，选择一张图片并保存。
7. 重新载入网页，确认自定义图标仍然存在。
8. 点击“恢复默认”，确认默认图标恢复。
