EVE Chat v0.9 — 动态回复与后台通知
===================================

本版基于 v0.8 已确认功能版继续完善，不加入关系值、好感度、信任度等系统。

一、部署
--------
1. 解压 ZIP。
2. 将 index.html、eve-sw.js 和整个 js 文件夹上传到 GitHub 仓库 EVE 根目录。
3. 保持结构：

   EVE/
   ├─ index.html
   ├─ eve-sw.js
   └─ js/
      ├─ weather.js
      ├─ proactive.js
      ├─ adapter.js
      ├─ memory.js
      ├─ timeline.js
      ├─ recall.js
      ├─ stickers.js
      ├─ moments.js
      ├─ notifications.js
      ├─ healthcheck.js
      └─ feature-settings.js

4. 上传后建议用带版本参数的网址打开：
   https://kanamememe.github.io/EVE/?v=0.9

二、本版新增入口
--------------
打开原本的“聊天设置 → EVE 扩展功能”，新增：

- 动态回复增强
- 后台通知

三、动态回复增强
--------------
- 修复用户评论角色动态后，角色回复经常生成失败的问题。
- 对 AI 返回的纯文本、JSON 数组、JSON 对象、Markdown 代码块进行兼容解析。
- 失败时可自动重试。
- 原生批量动态评论没有产生结果时，启用后备生成。
- 后备评论会读取角色人设、天气、时间、扩展记忆、共同时间线和近期聊天背景。
- 保留原有动态页面和发布流程，只对评论回复流程做增强。
- 评论区增加轻量微信式背景、间距与排版细节，不更换原页面整体风格。
- 增强失败时可选择回退到原生回复逻辑。

设置项：
- 修复角色回复用户评论
- 批量评论失败时自动后备生成
- 微信式评论区细节样式
- 动态互动通知
- 原生逻辑回退
- 重试次数
- 后备评论角色数量
- 评论长度限制

四、后台通知
------------
- 支持角色聊天新消息通知。
- 支持动态评论、动态回复通知。
- 支持主动聊天通知。
- 可桥接 EVE 原生通知事件。
- 可关闭通知正文预览。
- 可设置仅在页面不在前台时提醒。
- 包含通知权限申请和测试按钮。
- eve-sw.js 用于显示通知以及点击通知后重新聚焦网页。

重要限制：
- 此版本没有推送服务器。
- 网页被系统完全关闭后，eve-sw.js 不会自行调用 Gemini 或生成新消息。
- iPhone 将网页暂停时，只有已经发生并传给通知模块的事件才能显示通知；真正的远程推送仍需要后端。

五、原有功能仍保留
----------------
- 实时天气与角色当地时间
- AI 主动聊天
- 自动回复使用者消息
- 扩展记忆
- 共同时间线
- 使用者与角色消息撤回，并同步清理关联记忆和时间线
- 多张表情包导入、分类、标签、收藏、搜索、删除与拖曳排序
- 扩展健康检查

六、建议测试顺序
----------------
1. 确认原本聊天仍可正常使用。
2. 打开“扩展功能健康检查”。
3. 打开“动态回复增强”。
4. 在角色发布的动态下留言，等待角色回复。
5. 在“后台通知”中点击“申请通知权限”。
6. 点击“发送测试通知”。
7. 将网页切到后台，再测试聊天消息或动态互动通知。
