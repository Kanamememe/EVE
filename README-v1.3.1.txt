EVE Chat v1.3.1 — 回复失败静默恢复
=====================================

本版本以 v1.3 为基础，只完善角色回复失败的处理，不改变现有聊天、引用、表情包、记忆、时间线、行程和角色贴合功能。

新增文件
--------
js/response-recovery.js

功能
----
1. 角色回复失败时，不把失败占位消息显示在聊天界面。
2. 若本轮已经产生部分角色消息但随后失败，会撤销本轮未完成的部分消息。
3. 对临时网络错误、HTTP 408、429、5xx、空回复和格式异常，先在请求层静默重试。
4. 请求层仍未成功时，调用 EVE 原生智能回复流程，重新发出同一轮角色回复指令。
5. 用户在重试期间发送了新消息时，旧重试会自动取消，避免角色回复过时内容。
6. 清理被撤销失败消息所产生的扩展记忆、时间线和待确认记忆。
7. 旧聊天中已经保存的失败占位消息，可在启动时自动清理。
8. 明确的 Safety 拦截只隐藏失败消息并写入 AI 限制诊断，不会重复请求以绕过模型保护。

默认策略
--------
API 请求额外重试：1 次
回复指令重新发出：2 次
首次延迟：约 1.4 秒，并使用递增等待
最终仍失败：聊天区保持干净，只在顶部显示一次提示（可关闭）

设置位置
--------
聊天设置
→ EVE 扩展功能
→ 回复失败自动恢复

可调整：
- 总开关
- 自动重试
- API 请求重试次数
- 回复指令重试次数
- 首次重试延迟
- 是否清理历史失败消息
- 最终失败是否显示顶部提示

说明
----
这里的“后台”是指 EVE 页面或未来 App 仍处于运行状态时，在界面背后静默恢复。
若 Safari 已经完全关闭或 iOS 已暂停页面，前端无法继续调用 Gemini；未来封装为 App 后可再接入原生后台任务或服务器推送。

部署
----
解压后，将压缩包里的全部内容覆盖到 GitHub EVE 仓库根目录。
特别确认：
- js/response-recovery.js 存在
- index.html 已更新
- js/feature-settings.js 已更新
- js/healthcheck.js 已更新

部署后建议用：
https://kanamememe.github.io/EVE/?v=1.3.1
